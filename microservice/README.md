# Микросервис анализа эффективности использования аудиторного фонда

## 📋 Описание

Микросервис на FastAPI для построения отчетов по загруженности аудиторий университета с:
- **Кэшированием данных** - при повторном запросе того же периода данные берутся из кэша (24 часа)
- **Веб-интерфейсом** - интерактивные графики и таблицы
- **API endpoints** - для интеграции с другими системами

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install fastapi uvicorn pandas numpy sqlalchemy pymysql plotly jinja2 python-multipart
```

Или используйте requirements.txt:
```bash
pip install -r requirements.txt
```

### 2. Настройка конфигурации

Отредактируйте `config.json`:
```json
{
    "period": {
        "start_date": "2025-09-02",
        "end_date": "2025-12-01"
    },
    "excluded_buildings": [8, 9, 10],
    "db_config": {
        "user": "your_user",
        "password": "your_password",
        "host": "your_host",
        "database": "your_database"
    }
}
```

### 3. Запуск сервиса

```bash
cd microservice
python app.py
```

Сервис запустится на `http://localhost:8000`

## 📡 API Endpoints

### GET /api/reports
Получение полных данных отчета за период.

**Параметры:**
- `start_date` (string): Дата начала в формате YYYY-MM-DD
- `end_date` (string): Дата окончания в формате YYYY-MM-DD
- `force_refresh` (boolean): Принудительно обновить данные из БД

**Пример:**
```bash
curl "http://localhost:8000/api/reports?start_date=2025-09-02&end_date=2025-12-01"
```

**Ответ:**
```json
{
    "source": "cache",
    "period": {"start_date": "2025-09-02", "end_date": "2025-12-01"},
    "free_slots_count": 1500,
    "detailed_count": 3200,
    "free": [...],
    "detailed": [...],
    "pattern": [...]
}
```

### GET /api/reports/summary
Получение краткой сводки по периоду.

**Пример:**
```bash
curl "http://localhost:8000/api/reports/summary?start_date=2025-09-02&end_date=2025-12-01"
```

### GET /api/cache/status
Получение статуса кэша.

### DELETE /api/cache/clear
Очистка всего кэша.

## 💡 Как работает кэширование

1. При запросе данных вычисляется хэш от периода (start_date + end_date)
2. Если данные есть в кэше и не истекло время жизни (24 часа) → возвращаются из кэша
3. Если данных нет или `force_refresh=true` → загрузка из БД и сохранение в кэш

**Преимущества:**
- Повторные запросы того же периода выполняются мгновенно
- Снижение нагрузки на базу данных
- Возможность работы без БД после первичной загрузки

## 🔧 Исправления ошибок

### Проблема с типами данных
В оригинальных скриптах была проблема: при вводе строковых значений (например, "3") вместо чисел графики не строились.

**Решение в микросервисе:**
```python
# Все числовые колонки приводятся к numeric с обработкой ошибок
for col in numeric_cols_detailed:
    if col in df_occupied.columns:
        df_occupied[col] = pd.to_numeric(df_occupied[col], errors='coerce').fillna(0)
```

Это гарантирует, что:
- Строковые числа ("3") преобразуются в числа (3)
- Некорректные значения заменяются на 0
- Графики строятся корректно

## 🌐 Веб-интерфейс

Откройте `http://localhost:8000` в браузере для доступа к интерактивному интерфейсу:

1. **Выбор периода** - укажите даты начала и окончания
